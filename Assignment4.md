# Spreadsheets vs Databases (Health Data)

## 1. Difference between a Spreadsheet and a Database
- **Spreadsheet**: A file (e.g., Excel) where data is stored in rows and columns; best for small datasets, calculations, and quick analysis.
- **Database**: A structured system that stores large amounts of data in multiple related tables; designed for efficiency, accuracy, and multi-user access.
- **Key difference**: Spreadsheets are simple and flexible; databases are powerful and structured for large-scale data management.

## 2. Advantages & Disadvantages of Spreadsheets (Health Data)
- **Advantages**
  - **Easy to use**: No technical skills required.
  - **Good for small datasets**: Suitable for clinic patient logs.
  - **Built-in formulas & charts**: Fast calculations and visualization.
  - **Quick data entry & visualization**: Low setup overhead.
- **Disadvantages**
  - **Prone to human error**: Manual entry mistakes are common.
  - **Limited security**: Risky for sensitive health data.
  - **Not ideal for large datasets**: Performance and manageability issues.
  - **Hard for multiple users**: Concurrent editing is problematic.
  - **Poor data integrity**: Duplicates and inconsistencies easily occur.

## 3. Advantages & Disadvantages of Databases (Health Data)
- **Advantages**
  - **Handles large, complex datasets**: Efficient storage and retrieval.
  - **Strong security**: Important for patient confidentiality.
  - **Multi-user support**: Concurrent access and transactions.
  - **Maintains data integrity**: Less duplication, more accuracy.
  - **Relational capabilities**: Link related data (patients, visits, treatments).
- **Disadvantages**
  - **Requires technical knowledge**: Design and maintenance skills needed.
  - **Higher cost**: Setup and ongoing maintenance can be expensive.
  - **Less flexible for quick analysis**: Not as ad-hoc as spreadsheets.
  - **Needs structured design**: Schema and planning required before use.

## 4. Tasks Using a Spreadsheet
- **Creating charts/graphs**: e.g., infection rates.
- **Budget tracking**: Clinic finances.
- **Calculating averages**: e.g., patient age.
- **Simple data entry lists**: e.g., vaccination records.

## 5. Tasks Using a Database
- **Managing EHRs**: Electronic health records storage.
- **Tracking patient histories**: Across multiple visits.
- **Running complex queries**: e.g., patients with diabetes + hypertension.
- **Managing hospital systems**: Appointments, billing, and workflows.

## 6. Tasks Using BOTH Spreadsheet and Database
- **Data analysis**: Basic (spreadsheet) vs advanced (database).
- **Reporting**: Generate reports from DB, present in spreadsheet.
- **Data storage & export**: Store in DB, export subsets to spreadsheet.
- **Sorting & filtering**: Quick spreadsheet filters for DB extracts.

## 7. Tasks Using NEITHER
- **Performing surgery**: Requires human action.
- **Diagnosing patients directly**: Clinical judgment needed.
- **Conducting physical exams**: Hands-on medical evaluation.
- **Providing patient counseling**: Human interaction and empathy required.

## 8. Tasks Using Spreadsheet but NOT Database
- **Quick one-time calculations**
- **Instant simple charts**
- **Ad-hoc data exploration without setup**
- **Personal tracking**: e.g., daily calorie log

## 9. Tasks Using Database but NOT Spreadsheet
- **Managing millions of records**
- **Enforcing strict security rules**: HIPAA-level systems
- **Handling many simultaneous users**
- **Complex relational queries across tables**

## 10. (Repeat) Tasks Using BOTH Spreadsheet and Database
- **Data entry**
- **Basic analysis**
- **Sorting & filtering**
- **Generating reports**
